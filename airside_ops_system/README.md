# Entebbe International Airport - Airside Operations Digital Transformation System

Enterprise-grade Flask application for digitizing airside operations workflows and replacing paper forms in Apron Management, Apron Safety, and Airside Vehicle Control.

## 1. Overview

This project provides:

- Modular Flask architecture with blueprints for each operational domain.
- Role-based access control for admin, supervisor, inspector, operator, viewer, and auditor.
- Data model coverage for users, forms, inspections, permits, incidents, violations, stand allocations, handovers, and schedules.
- Digital form workflows for all required forms (1-25) with draft save, signatures, GPS fields, and attachment capability.
- PWA foundation with service worker and offline draft storage via IndexedDB.
- Dashboard KPIs and analytics endpoints with Chart.js integration.
- Export/PDF service layer for CSV, Excel, and form PDF generation.

## 2. Technology Stack

Backend:

- Flask 2.3+
- Flask-SQLAlchemy, Flask-Login, Flask-WTF, Flask-Migrate
- Flask-Caching, Flask-Limiter, Flask-SocketIO, Flask-Mail
- Celery + Redis (background workers)
- ReportLab, Pandas, OpenPyXL

Frontend:

- HTML/CSS/JS + Bootstrap 5.3
- jQuery, DataTables, Chart.js, Flatpickr, Select2
- Signature Pad, Leaflet, html2canvas, jsPDF, Dropzone

Database:

- SQLite (development default)
- PostgreSQL-ready via DATABASE_URL

## 4. Quick Start

### 4.1 Prerequisites

- Python 3.11+
- pip
- Redis (optional for Celery/background workloads)

### 4.2 Installation

```bash
cd airside_ops_system
python -m venv .venv
# Windows
.venv\Scripts\activate
# Linux/Mac
source .venv/bin/activate

pip install -r requirements.txt
```

### 4.3 Configuration

```bash
copy .env.example .env
```

Update key values in `.env`:

- `SECRET_KEY`
- `DATABASE_URL`
- `MAIL_*`
- `REDIS_URL`

### 4.4 Initialize Database

```bash
flask db init
flask db migrate -m "Initial schema"
flask db upgrade
flask seed-db
flask create-admin
```

### 4.5 Run Development Server

```bash
python run.py
```

Open: `http://127.0.0.1:5000`

Default admin (generated by CLI command):

- Username: `admin`
- Password: `Admin@2025!`

Change password immediately after first login.

## 7. Security Controls

Implemented controls:

- Password hashing via bcrypt
- Role and permission checks via decorators
- CSRF protection with Flask-WTF
- SQL injection mitigation via SQLAlchemy ORM
- Rate limiting via Flask-Limiter
- Session timeout configuration
- Audit trail table and helper API
- Optional two-factor auth structure (TOTP)

Recommended production hardening:

- HTTPS termination + HSTS
- Reverse proxy (Nginx)
- Strict CSP headers
- Malware scanning service for file uploads
- MFA enforcement for privileged roles
- Centralized SIEM logging

## 8. Background Tasks (Celery)

Use Celery for scheduled reports, notifications, and long-running PDF/Excel jobs.

Example worker startup:

```bash
celery -A app.celery_worker worker --loglevel=info
```

You can add `app/celery_worker.py` with your environment-specific broker setup.

## 9. Testing

Run tests:


## 10. Deployment Notes

- Set `FLASK_ENV=production`
- Use PostgreSQL (`DATABASE_URL=postgresql://...`)
- Set strong `SECRET_KEY` and `JWT_SECRET_KEY`
- Run behind Gunicorn + Nginx
- Enable Redis for cache/queue/rate limits

Example Gunicorn:

```bash
gunicorn -w 4 -k gevent -b 0.0.0.0:8000 run:app
```
