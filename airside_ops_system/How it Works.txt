Field Staff on iPad/Phone:
1. Opens TPBB form in app
2. Enters flight number → autocomplete from AODB
3. Enters bridge docking time (HH:MM)
4. Clicks "Save TPBB Record"
   ↓
App:
5. Saves form submission locally
6. Looks up flight in FlightMovement cache
7. Queues write-back: { flightId, BTI, time_value }
8. Returns to form with success message
   ↓
Background (every 5 min):
9. Scheduler picks up pending write-backs
10. Calls AodbClient.write_movement_time()
11. Updates queue item status → "completed"
12. On failure: retry up to 3 times, log error
   ↓
Admin/Supervisor:
13. Monitors queue via Admin → AODB Write-back Queue
14. Can manually trigger process or retry failed items
